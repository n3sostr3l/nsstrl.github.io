# Отчет по лабораторной работе 4
## Предсказание дефолта по кредиту (Бинарная классификация, борьба с дисбалансом классов, ROC-AUC, Stacking)

### 1. Выполненные действия

#### 1.1. Загрузка и предобработка данных
Обучающая и тестовая выборки были загружены из CSV-файлов. В ходе предварительного анализа (EDA) выявлено наличие пропущенных значений в признаках `MonthlyIncome` и `NumberOfDependents`. 
*   Пропуски были заполнены средними значениями по обучающей выборке (`fillna(mean)`).
*   Были удалены полные дубликаты строк (`drop_duplicates`).
*   **Борьба с дисбалансом классов:** Так как доля дефолтов в выборке составляет всего около 6-7%, был применен метод **Андерсемплинга (Undersampling)** с использованием алгоритмов `TomekLinks` и `ClusterCentroids` (библиотека `imblearn`) для выравнивания классов в обучающей выборке.

```python
from imblearn.under_sampling import TomekLinks
sampler = TomekLinks()
training_points_r, training_values_r = sampler.fit_resample(training_points, training_values)
```

#### 1.2. Обучение базовых моделей классификации
Были инициализированы и обучены на сбалансированных данных две базовые модели:
*   Логистическая регрессия (`LogisticRegression`)
*   Случайный лес (`RandomForestClassifier` с параметром `n_estimators=100`)

#### 1.3. Валидация, анализ матриц ошибок и подбор порога
Было доказано, что метрика **Accuracy (доля правильных ответов)** не подходит для данной задачи из-за сильного дисбаланса классов: константная модель, выдающая всегда "0" (нет дефолта), показала точность ~93.2%. 
Для оценки использовались **Матрицы ошибок (Confusion Matrix)** и **ROC-AUC**. Был исследован сдвиг порога классификации (0.3, 0.5, 0.7) для регулировки баланса между False Positive (выдали кредит потенциальному банкроту) и False Negative (отказали надежному клиенту).

#### 1.4. Самостоятельная работа: Обучение сложных алгоритмов и Ансамблирование
В попытке превзойти базовую модель Случайного леса были обучены дополнительные модели:
*   Метод опорных векторов (`SVC`)
*   Метод k-ближайших соседей (`KNeighborsClassifier` с `n_neighbors=50`)
*   **Стэкинг (StackingClassifier)** — мета-модель, объединившая в себе предсказания KNN, Random Forest и SVC, использующая Логистическую регрессию в качестве финального оценщика (final_estimator).

```python
from sklearn.ensemble import StackingClassifier
mega_ensemble = StackingClassifier(
    estimators=[("knn", knn), ("logreg", random_forest_model), ("svc", svc)]
)
mega_ensemble.fit(training_points_r, training_values_r)
```

### 5. Результаты тестирования (Метрика ROC-AUC)
Для сравнения использовалась площадь под ROC-кривой (ROC-AUC). Чем ближе значение к 1.0, тем лучше модель разделяет плохих и хороших заемщиков.

| Модель классификации | Значение ROC-AUC на тесте |
| :--- | :--- |
| **Логистическая регрессия (Базовая)** | 0.664 |
| **K-Nearest Neighbors (KNN)** | 0.593 |
| **Support Vector Classifier (SVC)** | 0.631 |
| **Стэкинг (Mega Ensemble: KNN+RF+SVC)** | 0.837 |
| **Случайный лес (Random Forest)** | **0.842** (Лучший результат) |

### 6. Заключение
1. **Метрики имеют значение:** Метрика Accuracy бесполезна при сильном дисбалансе классов. Главным инструментом оценки в кредитном скоринге является ROC-AUC и ручной выбор порога вероятности (Trade-off риска и прибыли).
2. **Превосходство древовидных алгоритмов:** Ансамбли на основе деревьев решений (`RandomForest`) показали себя значительно лучше линейных моделей и метрических алгоритмов (`SVC`, `KNN`), так как они менее чувствительны к выбросам и нелинейным зависимостям в данных (например, возраст или доход).
3. **Стэкинг не всегда панацея:** Сложный мета-алгоритм (`StackingClassifier`) показал отличный результат (0.837), но в данном конкретном запуске не смог обойти "чистый" Случайный Лес (0.842). Более того, использование SVC внутри стэкинга существенно увеличивает время обучения модели. Для продакшена в данном случае целесообразнее оставить одиночный оптимизированный Random Forest.
