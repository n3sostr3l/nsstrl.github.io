Отчет по лабораторной работе 5

Предсказание цен на недвижимость с использованием методов машинного обучения (Линейная регрессия, Случайный лес, AdaBoost)

1. Выполненные действия

1.1. Загрузка и предобработка данных

С помощью библиотеки pandas были загружены обучающая и тестовая выборки из файлов Excel. Проведена проверка на наличие пропусков (пропуски отсутствуют). Целевая переменная (Целевая.Цена) была отделена от остальных 15 признаков для обеих выборок.

import pandas as pd
target_variable_name = 'Целевая.Цена'
training_values = training_data[target_variable_name]
training_points = training_data.drop(columns=[target_variable_name])

1.2. Обучение базовых моделей

Были созданы и обучены две базовые регрессионные модели: Линейная регрессия (LinearRegression) и Случайный лес (RandomForestRegressor).

from sklearn import linear_model, ensemble
linear_regression_model = linear_model.LinearRegression()
random_forest_model = ensemble.RandomForestRegressor()

linear_regression_model.fit(training_points, training_values)
random_forest_model.fit(training_points, training_values)

1.3. Оценка важности признаков (Feature Selection)

На основе обученной модели случайного леса были выделены наиболее и наименее значимые признаки. * Топ-5 значимых: Жилая площадь (0.324), Оценка риелтора (0.275), Широта (0.172), Долгота (0.071), Год постройки (0.029). * Наименее значимые: Спальни, Состояние, Количество этажей, Год реновации (содержит подавляющее большинство нулей).

Для дальнейшей оптимизации был создан усеченный датасет (important_features), включающий только 7 самых значимых признаков.

1.4. Подбор гиперпараметров (RandomizedSearchCV)

Для улучшения модели случайного леса был применен случайный поиск по сетке гиперпараметров.

from sklearn.model_selection import RandomizedSearchCV
from numpy.random import randint

param_dist = {
    'n_estimators': [randint(50, 200)],
    'max_depth': [None, 10, 20, 40, 50],
    'min_samples_leaf': [1, 2, 4]
}
rscv = RandomizedSearchCV(ensemble.RandomForestRegressor(random_state=42), param_distributions=param_dist, n_iter=50, random_state=42)
rscv.fit(important_features, training_values)

Найденные лучшие параметры: max_depth=40, min_samples_leaf=2, n_estimators=85.

1.5. Применение мета-алгоритмов (AdaBoost)

В качестве самостоятельной работы была исследована модель бустинга AdaBoostRegressor, в которой в качестве базового оценщика (estimator) выступил RandomForestRegressor.

abr = ensemble.AdaBoostRegressor(ensemble.RandomForestRegressor())
abr.fit(important_features, training_values)

2. Организация данных

  • Обучающая выборка: 15 129 наблюдений, 15 признаков.
  • Тестовая выборка: 6 484 наблюдения.
  • Целевая переменная: Непрерывная величина (стоимость домов в округе Кинг, штат Вашингтон).
  • Метрики оценки: MAE (Средняя абсолютная ошибка) и RMSE (Корень из среднеквадратичной ошибки).

3. Как воспроизвести результаты

  1. Установить необходимые библиотеки: pip install pandas numpy matplotlib scikit-learn openpyxl.
  2. Поместить файлы predict_house_price_training_data.xlsx и predict_house_price_test_data.xlsx в корневую директорию проекта.
  3. Запустить все ячейки Jupyter Notebook (.ipynb) последовательно.

4. Анализ данных и гео-привязка

Анализ показал высокую значимость признаков Широта и Долгота. Визуальный и логический анализ карты округа Кинг подтвердил, что близость к центру Сиэтла (северная часть карты) экспоненциально увеличивает стоимость недвижимости, что модель успешно выявила самостоятельно.

5. Сравнение моделей и самостоятельная работа

В ходе самостоятельной работы модели обучались как на полном наборе данных (15 фичей), так и на усеченном (Топ-7 фичей). Исключение незначащих признаков позволило упростить модель, сохранив или даже незначительно улучшив её точность.

7. Заключение

  1. Базовые нелинейные ансамблевые методы (Случайный лес) показали себя значительно лучше Линейной регрессии (ошибка MAE снизилась почти в 2 раза: с ~126k до ~70k). Линейная модель сильно занижала цены на дорогую недвижимость.
  2. Очистка данных работает: Исключение "шумовых" и неинформативных признаков (таких как Год реновации, состоящий в основном из нулей) в связке с подбором параметров RandomizedSearchCV позволило получить наилучший результат (MAE: 70 489.92).
  3. Применение сложного ансамбля AdaBoost поверх Random Forest не дало прироста качества в конфигурации "из коробки" (ошибка слегка выросла). Это связано с тем, что Random Forest сам по себе является мощным ансамблем, и его помещение внутрь бустинга часто приводит к переобучению (overfitting) без тонкой ручной настройки скорости обучения (learning_rate) и глубины деревьев.
